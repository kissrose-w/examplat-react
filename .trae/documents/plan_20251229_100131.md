# 修复提交验证错误并优化OptionRenderer代码复用

## 问题分析
1. **验证冲突**：多个Form.Item使用相同的`name="answer"`，导致不同题型的验证规则冲突
2. **依赖项错误**：dependencies中使用了`type`，但Form中没有`type`字段
3. **访问权限问题**：validator中无法直接访问组件state中的options
4. **代码复用性低**：各题型渲染逻辑重复，可抽象通用组件

## 解决方案

### 1. 修复验证冲突
- 移除重复的Form.Item name="answer"，改为在Operation.tsx中统一处理答案
- 为不同题型创建独立的验证逻辑
- 简化OptionRenderer中的验证规则

### 2. 修复依赖项错误
- 移除无效的依赖项`type`
- 确保dependencies中只包含实际存在的Form字段

### 3. 优化验证逻辑
- 将选项内容验证移至选项输入时进行
- 简化答案验证，只检查是否选择了答案
- 移除validator中对组件state的直接访问

### 4. 提高代码复用性
- 抽象通用的选项渲染组件
- 统一单选题、多选题和判断题的渲染逻辑
- 提取公共的Form.Item配置

## 修改点

### 1. Operation.tsx
- 修改handleSubmit函数，统一处理不同题型的答案
- 添加题型字段到Form中，确保依赖项正确

### 2. OptionRenderer.tsx
- 移除重复的name="answer" Form.Item
- 简化验证规则，移除无法访问state的validator
- 抽象通用的选项渲染函数
- 统一处理不同题型的选项渲染

## 预期效果
1. 修复提交验证错误，表单验证正常工作
2. 提高代码复用性，减少重复代码
3. 简化验证逻辑，提高可维护性
4. 确保依赖项正确，验证规则按需触发
